# 远程控制台服务

## 1. 设计目标

通过标准 UDS `0x31 RoutineControl` 暴露一个“远程执行 shell 命令”的入口，用于：

- 现场调试
- 日志抓取
- 状态排查
- 简单维护命令执行

## 2. 实现方式

当前实现位于 `service/service_0x31_console.c`，核心思路是：

1. 创建一个虚拟字符设备；
2. 临时把 RT-Thread console / FinSH 输出切换到该设备；
3. 执行 MSH 命令；
4. 捕获输出文本；
5. 通过 `statusRecord` 返回给诊断客户端。

## 3. 请求约束

默认逻辑包含以下保护：

- 可要求扩展会话
- 可要求安全解锁
- 仅接受 `UDS_LEV_RCTP_STR`
- 仅接受指定 RID（默认 `0xF000`）

## 4. 响应内容

返回结果通常是：

```text
> ps
thread   pri status ...
...
```

也就是把执行命令和 shell 输出一起打包返回。

## 5. 风险与注意事项

### 5.1 安全风险

这是高权限能力。量产产品中建议至少做到：

- 仅扩展/编程会话可用
- 必须通过 0x27 解锁
- 命令白名单
- 输出长度限制
- 审计日志

### 5.2 输出截断

当前实现缓冲区大小由 `UDS_CONSOLE_BUF_SIZE` 控制；超出后会追加 `[TRUNCATED]` 标记。

### 5.3 与物理串口共存

如果开启 `UDS_CONSOLE_PASSTHROUGH`，输出会继续透传到旧 console；否则只在捕获缓冲中可见。
