# 迁移与裁剪说明

## 1. 先分清哪些是“核心”，哪些是“项目适配”

### 相对通用的部分

- `iso14229.c`
- `iso14229.h`
- `iso14229_rtt.c`
- `iso14229_rtt.h`
- 通用服务框架和事件分发

### 强项目相关的部分

- `service_0x22_0x2E_param.c`
- `service_0x31_console.c`
- `service_0x36_0x37_0x38_file.c`
- 示例中的 LED / ULOG 逻辑

## 2. 迁移到新 BSP 时优先检查

1. CAN 驱动和收发路径
2. RT-Thread console / FinSH 是否可切换
3. DFS / POSIX 文件接口是否可用
4. 线程优先级和内存大小
5. ULOG 是否启用

## 3. 常见裁剪策略

### 最小诊断集

- 0x10
- 0x11
- 0x3E
- 可选 0x22

### 产线参数集

- 0x10
- 0x22 / 0x2E
- 0x27
- 0x3E

### 远程维护集

- 0x10
- 0x27
- 0x2A
- 0x31
- 0x34~0x38
- 0x3E

## 4. 你大概率需要自己改的地方

- 安全算法
- DID 表设计
- 文件目录与权限控制
- shell 命令白名单
- IO 控制绑定
- 软件包宏命名统一

## 5. 建议的上线前检查项

- session timeout 行为是否符合预期
- 0x2F 被接管后的自动归还是否可靠
- 文件失败时是否会留下脏文件
- console 服务是否存在危险命令注入
- 高负载下 0x2A 是否会影响主业务线程
